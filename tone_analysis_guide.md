# 🎯 ฟีเจอร์ใหม่: การวิเคราะห์ Tone และแนวทางการแปล

## ✨ สิ่งที่เพิ่มใหม่

### 1. **การวิเคราะห์ลักษณะเอกสาร (Tone Analysis)**
- วิเคราะห์ว่าเอกสารเป็นแบบไหน: ทางการ, สบายๆ, เทคนิค, การตลาด, หรือทั่วไป
- ให้คะแนนความมั่นใจในการวิเคราะห์ (สูง/ปานกลาง/ต่ำ)

### 2. **แนวทางการแปล (Translation Suggestions)**
- เสนอแนวทางการแปลที่เหมาะสมกับ tone ของเอกสาร
- ให้ตัวอย่างคำแปลที่แนะนำ
- เคล็ดลับสำหรับการใช้ CAT Tools

---

## 📋 ประเภทเอกสารที่วิเคราะห์

### 1. 📋 เอกสารทางการ (Formal)
**ตัวบ่งชี้:**
- ใช้คำราชาศัพท์: กระผม, ดิฉัน, ท่าน
- มีคำสุภาพ: เรียน, ขอบคุณ, พิจารณา
- ประโยคยาว (>100 ตัวอักษร/segment)

**แนวทางแปล:**
- ใช้ภาษาทางการและสุภาพ
- รักษาโครงสร้างประโยค
- หลีกเลี่ยงคำพูดสบายๆ

### 2. 💬 เอกสารสบายๆ (Informal)
**ตัวบ่งชี้:**
- มีคำปิดท้าย: ครับ/ค่ะ, นะ, เหรอ
- ใช้ภาษาเป็นกันเอง

**แนวทางแปล:**
- ใช้ภาษาธรรมชาติและเข้าใจง่าย
- เพิ่มคำปิดท้ายตามบริบท
- รักษาโทนที่เป็นมิตร

### 3. 🔧 เอกสารทางเทคนิค (Technical)
**ตัวบ่งชี้:**
- มีคำศัพท์เทคนิค: ระบบ, ซอฟต์แวร์, ข้อมูล
- ใช้ภาษาอังกฤษมาก (>20%)
- มีตัวเลขและสถิติ

**แนวทางแปล:**
- คงคำศัพท์เทคนิคภาษาอังกฤษไว้
- ใช้คำศัพท์มาตรฐานในสายงาน
- รักษาความแม่นยำ

### 4. 📢 เนื้อหาการตลาด (Marketing)
**ตัวบ่งชี้:**
- มีคำชวนเชื่อ: พิเศษ, ฟรี, ลดราคา
- คำ adjective ที่เกินจริง
- Call-to-action ชัดเจน

**แนวทางแปล:**
- ปรับภาษาให้ดึงดูด
- รักษา call-to-action
- ปรับตามวัฒนธรรมเป้าหมาย

### 5. 📄 เอกสารทั่วไป (Neutral)
**เมื่อไม่ตรงเงื่อนไขข้างต้น**

## 🎨 การแสดงผลใน UI

### **ส่วนที่ 1: ประเภทเอกสาร**
```
🎯 การวิเคราะห์ลักษณะเอกสารและแนวทางการแปล

📋 เอกสารทางการ / Formal Document
เอกสารนี้มีลักษณะเป็นทางการ เหมาะสำหรับการแปลแบบเป็นทางการ
ความมั่นใจในการวิเคราะห์: สูง
```

### **ส่วนที่ 2: แนวทางการแปล**
- ✓ แนวทางหลักในการแปล
- ✓ สิ่งที่ควรทำและไม่ควรทำ
- ✓ ตัวอย่างคำแปลที่แนะนำ

### **ส่วนที่ 3: เคล็ดลับ CAT Tools**
- สร้าง Term Base ที่เหมาะสม
- ตั้ง QA rules
- ใช้ TM ที่เหมาะสม

### **ส่วนที่ 4: คะแนนวิเคราะห์**
- แสดง progress bar คะแนนแต่ละประเภท
- สถิติเพิ่มเติม (คำทางการ, คำสบายๆ, ฯลฯ)

## 💡 การใช้งาน

### **Step 1: อัปโหลดไฟล์**
- อัปโหลด PDF ตามปกติ

### **Step 2: ดูการวิเคราะห์**
- หลังจากประมวลผลเสร็จ จะเห็นส่วน **"การวิเคราะห์ลักษณะเอกสาร"**
- ดูประเภทเอกสารที่วิเคราะห์ได้

### **Step 3: ปรับแนวทางการแปล**
- อ่านแนวทางที่แนะนำ
- ดูตัวอย่างคำแปล
- ใช้เคล็ดลับสำหรับ CAT Tools

### **Step 4: ดาวน์โหลด CSV**
- ดาวน์โหลด Segments และ Glossary ตามปกติ
- นำไปใช้กับ CAT Tools พร้อมแนวทางที่แนะนำ
